# RELAIS

# Nos méthodes de travail

> **On prend le relais.**
>
> _Pour que vous puissiez avancer plus facilement._

---

# Pourquoi ce document existe

Une bonne entreprise ne repose pas uniquement sur de bonnes personnes.

Elle repose sur de bonnes méthodes.

Chez RELAIS, nous croyons que la qualité d'un produit est la conséquence directe de la qualité de la manière dont il est conçu.

Nos méthodes existent pour une raison simple :

**Produire un travail d'excellence, de manière reproductible.**

Peu importe l'équipe.

Peu importe le projet.

Peu importe le client.

Nous voulons que RELAIS travaille avec le même niveau d'exigence aujourd'hui comme dans dix ans.

---

# Notre philosophie de travail

Nous ne courons jamais après le développement.

Nous courons après la compréhension.

Car un problème bien compris conduit presque toujours à une bonne solution.

À l'inverse, une solution développée trop rapidement résout souvent le mauvais problème.

Nous préférons réfléchir une heure plutôt que corriger pendant une semaine.

---

# Notre méthode

Chaque projet suit toujours le même chemin.

```
Comprendre

↓

Observer

↓

Documenter

↓

Modéliser

↓

Concevoir

↓

Développer

↓

Tester

↓

Déployer

↓

Mesurer

↓

Améliorer
```

Nous ne sautons jamais une étape.

Chaque étape prépare la suivante.

---

# 1. Comprendre

Avant toute chose, nous cherchons à comprendre le métier de notre client.

Nous voulons savoir :

- Comment travaille-t-il aujourd'hui ?
- Quels sont ses objectifs ?
- Quelles sont ses frustrations ?
- Qu'est-ce qui lui fait perdre du temps ?
- Qu'est-ce qui fonctionne déjà ?

Nous ne proposons jamais une solution avant de comprendre le problème.

---

# 2. Observer

Les utilisateurs disent parfois une chose.

Leur quotidien en révèle une autre.

Nous observons les processus réels.

Nous cherchons à comprendre ce qui se passe sur le terrain.

Les meilleures idées viennent souvent de l'observation.

---

# 3. Documenter

Chez RELAIS,

une idée n'existe réellement que lorsqu'elle est écrite.

Nous documentons :

- les besoins ;
- les contraintes ;
- les décisions ;
- les hypothèses ;
- les processus ;
- les règles métier.

La documentation est un investissement.

Elle réduit les erreurs.

Elle facilite la transmission.

Elle accélère les projets.

---

# 4. Modéliser

Avant de penser à une base de données,

nous pensons au métier.

Nous identifions :

- les entités ;
- les responsabilités ;
- les relations ;
- les règles métier.

Nous construisons un modèle du monde réel avant de construire un modèle informatique.

---

# 5. Concevoir

Une fois le métier compris,

nous concevons l'architecture.

Nous choisissons :

- les technologies ;
- les responsabilités ;
- les interfaces ;
- les flux ;
- les permissions ;
- les intégrations.

Chaque décision doit rester simple, cohérente et durable.

---

# 6. Développer

Le développement n'est pas une phase d'improvisation.

Il consiste à transformer une conception validée en logiciel.

Le code doit être :

- lisible ;
- maintenable ;
- documenté lorsque nécessaire ;
- testé ;
- cohérent avec les standards RELAIS.

Nous écrivons du code pour les développeurs qui viendront après nous.

---

# 7. Tester

Un logiciel terminé n'est pas forcément un logiciel prêt.

Nous vérifions :

- le fonctionnement ;
- les cas limites ;
- les performances ;
- la sécurité ;
- l'expérience utilisateur.

Chaque anomalie corrigée avant la livraison renforce la confiance du client.

---

# 8. Déployer

Un déploiement doit être prévisible.

Jamais improvisé.

Chaque mise en production doit pouvoir être reproduite.

Nous privilégions des procédures simples, documentées et vérifiables.

---

# 9. Mesurer

Une fonctionnalité livrée n'est pas la fin du travail.

Nous observons :

- son utilisation ;
- ses performances ;
- les retours utilisateurs ;
- les difficultés rencontrées.

Les données guident les améliorations.

---

# 10. Améliorer

Aucun produit n'est parfait.

Nous améliorons continuellement :

- nos logiciels ;
- nos processus ;
- notre documentation ;
- notre communication ;
- notre manière de travailler.

L'amélioration continue fait partie de notre identité.

---

# Notre manière de communiquer

Nous privilégions toujours :

- la clarté ;
- la transparence ;
- le respect ;
- la simplicité.

Nous critiquons les idées.

Jamais les personnes.

Les désaccords sont normaux.

Ils deviennent une richesse lorsqu'ils sont exprimés avec respect.

---

# Notre manière de collaborer

Nous travaillons en équipe.

Nous partageons nos connaissances.

Nous documentons notre travail.

Nous aidons avant qu'on nous le demande.

Le succès individuel est apprécié.

Le succès collectif est recherché.

---

# Notre définition de l'excellence

Pour RELAIS,

l'excellence ne consiste pas à produire le logiciel le plus complexe.

Elle consiste à produire la meilleure solution possible pour notre client.

Une solution :

- simple ;
- fiable ;
- élégante ;
- durable ;
- comprise par son utilisateur.

L'excellence est un standard.

Pas un objectif occasionnel.

---

# Notre engagement

Nos méthodes évoluent avec notre expérience.

Mais leur objectif restera toujours le même.

Créer des produits de qualité.

Construire des relations durables.

Faire progresser nos équipes.

Et permettre à nos clients d'avancer plus facilement.

Car chez RELAIS,

nous ne construisons pas seulement des logiciels.

Nous construisons une manière de travailler.
