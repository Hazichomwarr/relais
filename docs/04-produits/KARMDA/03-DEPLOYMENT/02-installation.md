# KARMDA

# 01 - Installation

## Déploiement RELAIS

> Ce document fait partie du programme officiel de déploiement KARMDA.

**Objectif**

À la fin de cette lecture, vous serez capable de préparer et configurer correctement un nouvel établissement avant son utilisation de KARMDA.

**Temps estimé**

20 minutes

**Prérequis**

README.md

---

# Pourquoi ce chapitre est important

L'installation constitue la base de tout le projet.

Une mauvaise configuration initiale peut entraîner des erreurs, des pertes de temps et une mauvaise expérience pour les utilisateurs.

À l'inverse, une installation rigoureuse facilite toutes les étapes suivantes : onboarding, formation, mise en production et support.

Chez RELAIS, nous considérons que le temps investi dans la préparation est toujours rentabilisé.

---

# Notre philosophie

Installer KARMDA ne consiste pas uniquement à créer une école dans le système.

Il s'agit de préparer un environnement de travail fiable, cohérent et sécurisé.

Chaque paramètre configuré aujourd'hui permettra aux utilisateurs de travailler sereinement demain.

---

# Avant de commencer

Avant toute intervention, assurez-vous de disposer des informations suivantes.

## Informations sur l'établissement

- nom officiel ;
- adresse ;
- téléphone ;
- adresse e-mail ;
- devise utilisée ;
- fuseau horaire ;
- année scolaire en cours.

---

## Organisation pédagogique

- niveaux d'enseignement ;
- classes ;
- sections ;
- cycles ;
- calendrier scolaire.

---

## Organisation administrative

- direction ;
- personnel administratif ;
- enseignants ;
- politique de gestion des utilisateurs.

---

# Création de l'établissement

Créer le nouvel établissement dans KARMDA.

Vérifier immédiatement :

- le nom affiché ;
- les coordonnées ;
- les paramètres régionaux ;
- la devise ;
- le fuseau horaire.

Une erreur à cette étape sera visible dans tout le système.

---

# Configuration initiale

Configurer les paramètres principaux.

Par exemple :

- année scolaire active ;
- périodes académiques ;
- système de notation ;
- politique des présences ;
- paramètres financiers ;
- préférences de communication.

Toutes ces informations doivent être validées avec le responsable de l'établissement.

---

# Création des premiers comptes

Créer les premiers utilisateurs.

En priorité :

- administrateur principal ;
- direction ;
- administration.

Ces comptes permettront ensuite de gérer les autres utilisateurs.

---

# Vérification des accès

Avant de poursuivre, vérifier que chaque utilisateur peut :

- se connecter ;
- accéder à son tableau de bord ;
- consulter les informations correspondant à son rôle.

Un accès fonctionnel est préférable à plusieurs comptes non testés.

---

# Vérifications techniques

Avant de poursuivre le déploiement, confirmer que :

- l'application est accessible ;
- les performances sont satisfaisantes ;
- les sauvegardes sont opérationnelles ;
- les notifications fonctionnent ;
- les paramètres de sécurité sont correctement appliqués.

---

# Erreurs fréquentes

Les difficultés les plus courantes proviennent de :

- paramètres régionaux incorrects ;
- mauvaise année scolaire active ;
- rôles mal attribués ;
- informations administratives incomplètes ;
- absence de vérification après configuration.

La majorité de ces problèmes peuvent être évités grâce à une simple relecture.

---

# Bonnes pratiques RELAIS

✔ Configurer lentement.

✔ Vérifier chaque étape.

✔ Faire valider les informations importantes par l'établissement.

✔ Ne jamais supposer qu'une information est correcte.

✔ Documenter toute configuration particulière.

---

# Ce qu'il faut retenir

À l'issue de ce chapitre, retenez les idées suivantes :

- une installation prépare tout le reste du projet ;
- chaque configuration doit être validée ;
- les premiers comptes doivent être testés ;
- une erreur au départ peut impacter tout l'établissement ;
- la qualité du déploiement dépend de la qualité de la préparation.

---

## Phrase à retenir

> **"Une installation réussie est celle qui devient invisible pour les utilisateurs."**
