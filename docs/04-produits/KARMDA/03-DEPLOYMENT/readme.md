# KARMDA

# Déploiement

Bienvenue dans la documentation officielle de déploiement de KARMDA.

Cette documentation accompagne les équipes RELAIS chargées de mettre en œuvre KARMDA auprès des établissements scolaires.

Son objectif est de garantir que chaque déploiement soit réalisé avec la même qualité, la même rigueur et le même niveau d'accompagnement.

---

# Notre philosophie

Chez RELAIS, le déploiement ne consiste pas à installer un logiciel.

Il consiste à accompagner un établissement dans une nouvelle manière de travailler.

Un logiciel peut être installé en quelques heures.

L'adoption d'une nouvelle organisation demande de l'écoute, de la préparation et de l'accompagnement.

Notre mission est de rendre cette transition aussi simple que possible.

---

# Pourquoi ce dossier est important

La qualité d'un déploiement influence directement la satisfaction du client.

Même le meilleur logiciel peut être mal perçu si sa mise en œuvre est mal préparée.

À l'inverse, un établissement bien accompagné développera rapidement la confiance nécessaire pour utiliser pleinement KARMDA.

Chaque déploiement est donc une étape essentielle de notre relation avec nos clients.

---

# À qui s'adresse cette documentation ?

Cette documentation est destinée à toute personne participant au déploiement de KARMDA, notamment :

- les chefs de projet ;
- les équipes techniques ;
- les équipes d'implémentation ;
- les formateurs ;
- les responsables du support ;
- toute personne amenée à accompagner un établissement durant sa mise en service.

---

# Les objectifs d'un déploiement

Un déploiement réussi ne signifie pas simplement que KARMDA fonctionne.

Il signifie que :

- l'établissement est correctement configuré ;
- les données sont fiables ;
- les utilisateurs disposent de leurs accès ;
- les équipes comprennent leur nouvel environnement de travail ;
- l'établissement est prêt à utiliser KARMDA en toute confiance.

---

# Les principes RELAIS

Chaque déploiement repose sur les principes suivants.

## Préparer avant d'agir

Un déploiement bien préparé évite la majorité des difficultés.

---

## Communiquer en permanence

Le client doit toujours savoir :

- où en est le projet ;
- quelles sont les prochaines étapes ;
- ce que nous attendons de lui.

Une bonne communication réduit les incertitudes.

---

## Vérifier avant de livrer

Chaque configuration doit être contrôlée.

Chaque donnée importante doit être validée.

Chaque utilisateur doit pouvoir accéder à son espace.

Nous ne supposons jamais qu'une étape est terminée.

Nous la vérifions.

---

## Former avant de lancer

Un établissement qui ne comprend pas son nouvel outil ne pourra pas l'utiliser sereinement.

La formation fait partie intégrante du déploiement.

Elle n'est jamais une option.

---

## Accompagner après le lancement

La mise en production n'est pas la fin du projet.

C'est le début de l'utilisation quotidienne de KARMDA.

Les premières semaines sont essentielles.

Nous restons disponibles pour accompagner l'établissement dans cette nouvelle étape.

# Qu'est-ce qu'un déploiement réussi ?

Chez RELAIS, un déploiement est considéré comme réussi lorsque :

- l'établissement utilise KARMDA de manière autonome ;
- les utilisateurs ont confiance dans la plateforme ;
- les données sont fiables ;
- les équipes adoptent progressivement leurs nouveaux processus de travail ;
- le client considère RELAIS comme un partenaire de confiance.

Nous ne mesurons pas notre réussite au nombre d'établissements installés.

Nous la mesurons au nombre d'établissements qui utilisent réellement KARMDA avec sérénité.
