# KARMDA

# 02 - Formation des administrateurs

## Formation RELAIS

> Ce document présente le programme officiel de formation des administrateurs et du personnel administratif utilisant KARMDA.

**Objectif**

À la fin de cette formation, l’administrateur sera capable d’accéder à son espace, de gérer les principales données administratives, d’effectuer les opérations liées aux élèves et aux classes, de vérifier son travail et de signaler correctement une difficulté.

**Durée indicative**

2 à 4 heures selon :

- le périmètre activé ;
- le niveau des participants ;
- le nombre d’exercices ;
- la complexité des processus de l’établissement.

**Public concerné**

- administrateurs KARMDA ;
- personnel administratif ;
- secrétaires ;
- responsables des inscriptions ;
- responsables de la vie scolaire ;
- référents internes ;
- direction, lorsque certaines fonctions administratives lui sont attribuées.

**Prérequis**

- compte KARMDA actif ;
- rôle administratif correctement configuré ;
- appareil fonctionnel ;
- accès Internet ;
- connaissance des procédures administratives de l’établissement ;
- participation à l’onboarding du projet ou présentation équivalente.

---

# Pourquoi cette formation est importante

Les administrateurs jouent un rôle central dans la fiabilité de KARMDA.

Ils interviennent généralement sur des informations utilisées par plusieurs autres personnes :

- direction ;
- enseignants ;
- parents ;
- élèves ;
- service financier ;
- équipe RELAIS.

Une erreur administrative peut donc produire plusieurs conséquences.

Par exemple :

- un élève placé dans la mauvaise classe n’apparaît pas dans la bonne liste de présence ;
- un doublon crée deux dossiers pour la même personne ;
- un parent lié au mauvais élève consulte des informations qui ne le concernent pas ;
- une mauvaise année scolaire fausse les inscriptions ;
- un utilisateur possédant trop de permissions accède à des données sensibles ;
- une information non vérifiée apparaît dans les rapports.

L’administrateur contribue directement à la qualité générale de la plateforme.

---

# Notre philosophie

Le rôle d’un administrateur ne consiste pas à remplir KARMDA aussi rapidement que possible.

Il consiste à enregistrer les bonnes informations, dans le bon dossier, au bon moment, avec le bon niveau de contrôle.

> **Une opération administrative rapide mais incorrecte coûte souvent plus de temps qu’une opération réalisée lentement et vérifiée.**

La formation doit donc développer cinq habitudes :

1. identifier correctement la personne ou le dossier ;
2. vérifier avant de créer ;
3. respecter les règles de l’établissement ;
4. contrôler après l’enregistrement ;
5. documenter les corrections importantes.

---

# Résultats attendus

À la fin de la formation, l’administrateur doit pouvoir :

- se connecter avec son propre compte ;
- comprendre les limites de son rôle ;
- naviguer dans l’espace administratif ;
- retrouver un élève existant ;
- éviter la création d’un doublon ;
- créer ou compléter un dossier ;
- inscrire un élève dans la bonne année scolaire ;
- affecter un élève à la bonne classe ;
- consulter les listes principales ;
- mettre à jour une information autorisée ;
- identifier une donnée nécessitant une validation ;
- vérifier le résultat de son opération ;
- respecter les règles de confidentialité ;
- signaler un problème de manière exploitable.

Selon le périmètre, il peut également devoir :

- gérer les responsables légaux ;
- gérer les utilisateurs ;
- gérer les enseignants ;
- suivre les documents administratifs ;
- consulter les présences ;
- publier des communications ;
- enregistrer certaines informations financières ;
- générer des rapports.

# 1. Comprendre le rôle de l’administrateur

Avant de montrer l’interface, le formateur doit expliquer ce que l’établissement attend de l’administrateur.

L’administrateur peut être responsable de :

- la qualité des dossiers ;
- l’enregistrement des inscriptions ;
- la mise à jour des informations ;
- la création de certains comptes ;
- la gestion des classes ;
- la vérification des listes ;
- la correction des anomalies ;
- l’accompagnement de certains utilisateurs ;
- la transmission des problèmes à RELAIS.

Toutes ces responsabilités ne doivent pas être attribuées automatiquement à une seule personne.

Elles doivent correspondre :

- au poste réel ;
- aux permissions ;
- à l’organisation interne ;
- au niveau d’autorité ;
- au périmètre du déploiement.

---

# Ce que l’administrateur ne doit pas faire

Selon son rôle, l’administrateur ne doit pas nécessairement :

- modifier les règles académiques ;
- changer les notes des enseignants ;
- modifier des montants financiers sans autorisation ;
- attribuer des permissions supérieures ;
- consulter des informations confidentielles hors de ses responsabilités ;
- supprimer des dossiers sans validation ;
- contourner les procédures ;
- utiliser le compte d’une autre personne ;
- prendre seul des décisions administratives qui relèvent de la direction.

Une permission technique ne remplace pas une autorisation professionnelle.

> **Pouvoir réaliser une action dans le système ne signifie pas toujours être autorisé à la réaliser dans l’établissement.**

---

# 2. Se connecter à KARMDA

## Accéder à la plateforme

Le participant doit savoir :

- ouvrir le navigateur approprié ;
- accéder à l’adresse officielle de KARMDA ;
- reconnaître la page de connexion ;
- saisir son identifiant ;
- saisir son mot de passe ;
- confirmer qu’il se trouve dans le bon établissement.

Le formateur doit rappeler de ne pas utiliser une adresse reçue dans un message non vérifié lorsque son origine semble douteuse.

---

# Première connexion

Lors de la première connexion, l’utilisateur peut devoir :

- utiliser un mot de passe temporaire ;
- définir un nouveau mot de passe ;
- confirmer certaines informations ;
- accepter les conditions prévues ;
- vérifier son profil.

Le mot de passe doit être :

- personnel ;
- suffisamment difficile à deviner ;
- différent de ceux partagés avec d’autres personnes ;
- conservé de manière sécurisée.

---

# Protéger son compte

L’administrateur doit :

- ne jamais partager son mot de passe ;
- ne pas laisser un collègue utiliser son compte ;
- fermer sa session sur un appareil partagé ;
- verrouiller son appareil lorsqu’il s’absente ;
- signaler un accès suspect ;
- utiliser uniquement les canaux approuvés ;
- éviter d’enregistrer son mot de passe sur un appareil public ;
- demander une réinitialisation lorsqu’il ne se souvient plus du mot de passe.

## Pourquoi le partage de compte est dangereux

Lorsqu’un compte est partagé, il devient difficile de savoir :

- qui a créé une donnée ;
- qui a réalisé une correction ;
- qui a supprimé une information ;
- qui a consulté un dossier ;
- qui est responsable d’une erreur.

Chaque utilisateur doit utiliser son propre compte.

---

# Exercice pratique 1

Chaque participant doit :

1. accéder à KARMDA ;
2. se connecter avec son propre compte ;
3. vérifier son nom ;
4. confirmer son rôle ;
5. identifier l’établissement ;
6. se déconnecter ;
7. se reconnecter.

## Critères de réussite

- le participant utilise son propre compte ;
- il se trouve dans le bon établissement ;
- il connaît la méthode de déconnexion ;
- il sait quoi faire en cas de mot de passe oublié.

---

# 3. Comprendre l’espace administratif

Le formateur doit présenter uniquement les sections utiles au rôle concerné.

L’espace peut inclure :

- tableau de bord ;
- élèves ;
- inscriptions ;
- classes ;
- enseignants ;
- utilisateurs ;
- parents ou responsables ;
- présences ;
- résultats ;
- finance ;
- communications ;
- rapports ;
- paramètres.

La présence d’un menu dépend :

- du rôle ;
- des modules ;
- des permissions ;
- de la configuration de l’établissement.

Deux administrateurs peuvent donc voir des espaces différents.

---

# Lire le tableau de bord

Le tableau de bord peut présenter :

- le nombre d’élèves ;
- les classes ;
- les inscriptions ;
- les absences ;
- les tâches en attente ;
- les activités récentes ;
- les indicateurs administratifs ;
- les alertes.

L’administrateur doit comprendre :

- la période concernée ;
- les filtres actifs ;
- la date de mise à jour ;
- le sens des indicateurs ;
- les limites des données affichées.

Un chiffre affiché ne doit pas être interprété sans vérifier son contexte.

---

# Utiliser les menus et filtres

Le participant doit savoir :

- ouvrir une section ;
- revenir à la liste principale ;
- rechercher ;
- filtrer ;
- trier ;
- ouvrir un dossier ;
- distinguer consultation et modification ;
- reconnaître une action destructive ;
- fermer ou annuler une action.

Le formateur doit expliquer que les filtres peuvent masquer certaines données.

Si un dossier semble absent, vérifier :

- la recherche active ;
- les filtres ;
- l’année scolaire ;
- le statut ;
- la classe ;
- la pagination.

---

# Exercice pratique 2

Le participant doit :

1. ouvrir la liste des élèves ;
2. appliquer un filtre de classe ;
3. rechercher un élève ;
4. ouvrir son dossier ;
5. revenir à la liste ;
6. retirer le filtre ;
7. confirmer que le nombre de résultats change.

---

# 4. Rechercher avant de créer

L’une des règles les plus importantes pour un administrateur est :

> **Toujours rechercher avant de créer.**

Créer immédiatement un nouveau dossier peut produire un doublon.

---

# Pourquoi les doublons apparaissent

Les doublons peuvent être causés par :

- une variation d’orthographe ;
- un nom incomplet ;
- un matricule non recherché ;
- une ancienne inscription ;
- une mauvaise année scolaire ;
- un dossier créé par un autre administrateur ;
- une recherche trop rapide ;
- un filtre resté actif.

## Exemple

```text
Mariam Ouédraogo
Mariam Ouedraogo
Mariam A. Ouédraogo
```

Ces résultats peuvent désigner la même élève.

---

# Comment rechercher efficacement

Avant de créer un élève, rechercher avec :

- le matricule ;
- le nom ;
- le prénom ;
- la date de naissance ;
- le téléphone du responsable ;
- l’ancienne classe ;
- l’année scolaire précédente.

Utiliser plusieurs informations lorsque le nom est courant.

---

# Que faire en cas de doute

Lorsqu’un dossier ressemble à la personne recherchée :

1. ouvrir le dossier ;
2. comparer les informations ;
3. consulter les inscriptions ;
4. vérifier les responsables ;
5. demander confirmation ;
6. ne pas créer un nouveau dossier avant clarification.

L’administrateur ne doit pas fusionner ou supprimer seul deux dossiers sans autorisation.

---

# Exercice pratique 3

Scénario :

> Une mère se présente pour inscrire sa fille, Mariam Ouédraogo. Vous trouvez deux résultats portant un nom proche. Déterminez si l’un des dossiers correspond à l’élève avant de créer une nouvelle personne.

Le participant doit expliquer :

- les informations comparées ;
- sa décision ;
- la validation nécessaire ;
- le risque évité.

---

# 5. Gérer le dossier d’un élève

Le dossier d’un élève peut contenir :

- identité ;
- date et lieu de naissance ;
- sexe ;
- matricule ;
- photo ;
- coordonnées ;
- responsables légaux ;
- informations d’urgence ;
- année scolaire ;
- classe ;
- inscriptions ;
- statut ;
- documents ;
- informations académiques ;
- informations financières selon les permissions.

Tous les champs ne sont pas nécessairement modifiables par tous les administrateurs.

---

# Créer un nouvel élève

Avant la création, vérifier :

- qu’aucun dossier n’existe ;
- que les informations officielles sont disponibles ;
- que les champs obligatoires sont connus ;
- que la personne possède l’autorisation nécessaire ;
- que la bonne année scolaire est active.

## Étapes générales

1. ouvrir la section Élèves ;
2. rechercher l’élève ;
3. confirmer l’absence de dossier ;
4. sélectionner l’action de création ;
5. saisir les informations obligatoires ;
6. vérifier l’orthographe ;
7. enregistrer ;
8. consulter le dossier créé ;
9. vérifier l’identifiant ;
10. poursuivre l’inscription si nécessaire.

---

# Saisir correctement l’identité

L’identité doit correspondre aux documents officiels utilisés par l’établissement.

Vérifier :

- prénom ;
- nom ;
- autres prénoms ;
- accents ;
- ordre des noms ;
- date de naissance ;
- sexe ;
- matricule.

Ne pas :

- remplacer un nom par un surnom ;
- inventer une date ;
- attribuer un matricule non autorisé ;
- ajouter des titres dans le champ du nom ;
- corriger une identité officielle selon une supposition personnelle.

---

# Gérer les informations manquantes

Lorsqu’une information manque :

- vérifier si elle est obligatoire ;
- demander la source appropriée ;
- laisser vide lorsque le système l’autorise ;
- utiliser le statut prévu ;
- ajouter une remarque lorsque le processus le permet ;
- suivre l’information à compléter.

Ne jamais inventer une valeur pour terminer rapidement le dossier.

---

# Vérifier après création

Après l’enregistrement, vérifier :

- le nom ;
- le matricule ;
- la date de naissance ;
- l’année scolaire ;
- la classe éventuelle ;
- le statut ;
- les responsables ;
- le message de confirmation.

> **Un dossier créé n’est pas encore un dossier correct. Il devient correct après vérification.**

---

# Modifier un dossier

Avant toute modification :

- confirmer l’identité ;
- vérifier son autorisation ;
- comprendre la raison ;
- identifier la source ;
- vérifier l’impact ;
- documenter les changements importants.

## Modifications simples

Peuvent inclure :

- correction d’un téléphone ;
- ajout d’une adresse ;
- ajout d’un contact ;
- correction d’une faute évidente validée.

## Modifications sensibles

Peuvent inclure :

- nom officiel ;
- date de naissance ;
- responsable légal ;
- matricule ;
- statut administratif ;
- changement de classe ;
- suppression ;
- transfert.

Ces changements peuvent nécessiter :

- un document ;
- une validation ;
- une trace ;
- l’intervention d’un responsable supérieur.

---

# Exercice pratique 4

Créer un dossier de test en respectant les informations fournies.

Le participant doit :

1. rechercher le nom ;
2. créer le dossier ;
3. saisir les champs obligatoires ;
4. enregistrer ;
5. rouvrir le dossier ;
6. vérifier chaque donnée ;
7. expliquer quelles informations ne pourraient pas être inventées.

---

# 6. Gérer les responsables légaux

Un responsable légal peut être :

- parent ;
- tuteur ;
- membre de la famille ;
- représentant autorisé ;
- personne à contacter selon les règles de l’établissement.

L’administrateur doit distinguer :

- la personne responsable ;
- la personne à contacter ;
- la personne autorisée à consulter les informations ;
- la personne autorisée à récupérer l’élève ;
- le payeur, lorsque le processus financier le prévoit.

Ces rôles peuvent être différents.

---

# Rechercher avant de créer un responsable

Un parent peut avoir plusieurs enfants dans le même établissement.

Il ne faut pas nécessairement créer un nouveau responsable pour chaque enfant.

Avant de créer :

- rechercher le téléphone ;
- rechercher l’adresse e-mail ;
- rechercher le nom ;
- vérifier les enfants déjà associés.

---

# Lier le bon responsable au bon élève

Avant validation, vérifier :

- l’identité du responsable ;
- la relation ;
- les coordonnées ;
- l’élève concerné ;
- les droits d’accès ;
- la source de l’information.

Une mauvaise relation peut exposer les données d’un enfant à une personne non autorisée.

---

# Coordonnées partagées

Plusieurs responsables peuvent parfois utiliser :

- le même téléphone ;
- la même adresse e-mail ;
- la même adresse.

Cela ne signifie pas automatiquement qu’ils représentent une seule personne.

La structure doit respecter les règles réelles de l’établissement et de KARMDA.

---

# Exercice pratique 5

Scénario :

> Un père possède déjà un compte pour son premier enfant. Son deuxième enfant vient d’être inscrit.

Le participant doit :

1. rechercher le responsable ;
2. confirmer son identité ;
3. éviter de créer un doublon ;
4. lier le deuxième enfant ;
5. vérifier que seules les bonnes relations apparaissent.

---

# 7. Gérer les inscriptions

Un dossier d’élève et une inscription ne représentent pas nécessairement la même chose.

## Élève

Représente la personne.

## Inscription

Représente sa participation à une année scolaire, un niveau ou une classe donnée.

Un élève peut donc posséder :

- un seul dossier ;
- plusieurs inscriptions au fil des années.

Cette distinction est importante.

Créer un nouveau dossier chaque année produit des doublons.

---

# Avant d’inscrire

Vérifier :

- l’identité ;
- l’année scolaire ;
- le niveau ;
- la classe ;
- le statut ;
- la capacité ou règle de l’établissement ;
- les documents nécessaires ;
- les validations éventuelles.

---

# Réaliser une inscription

Étapes générales :

1. ouvrir le dossier de l’élève ;
2. consulter les inscriptions existantes ;
3. vérifier l’année scolaire ;
4. lancer la nouvelle inscription ;
5. sélectionner le niveau ;
6. sélectionner la classe ;
7. compléter les informations ;
8. enregistrer ;
9. vérifier que l’élève apparaît dans la bonne classe.

---

# Éviter les erreurs d’inscription

Erreurs fréquentes :

- mauvaise année scolaire ;
- mauvaise classe ;
- inscription en double ;
- mauvaise section ;
- ancien statut non mis à jour ;
- élève créé une deuxième fois ;
- inscription sans validation nécessaire.

---

# Changer un élève de classe

Un changement de classe peut affecter :

- les présences ;
- les listes ;
- les évaluations ;
- les notes ;
- les enseignants ;
- les communications ;
- les rapports.

Avant le changement :

- confirmer la décision ;
- vérifier sa date d’effet ;
- comprendre le traitement des données précédentes ;
- informer les responsables concernés ;
- documenter la modification.

Il ne faut pas déplacer un élève uniquement parce qu’un utilisateur le demande oralement sans autorisation.

---

# Retrait, transfert ou départ

Lorsqu’un élève quitte l’établissement, le processus peut exiger :

- un statut de transfert ;
- une date de départ ;
- une raison ;
- un document ;
- une validation ;
- la conservation de l’historique.

Éviter de supprimer le dossier pour représenter un départ.

La suppression peut faire disparaître l’historique nécessaire.

---

# Exercice pratique 6

Scénario :

> Un élève déjà présent dans KARMDA revient pour une nouvelle année scolaire.

Le participant doit :

1. rechercher son dossier ;
2. consulter son ancienne inscription ;
3. créer la nouvelle inscription ;
4. sélectionner la bonne classe ;
5. vérifier l’absence de nouveau doublon ;
6. confirmer les deux inscriptions dans l’historique.

---

# 8. Gérer les classes et les structures

Selon ses permissions, l’administrateur peut consulter ou gérer :

- cycles ;
- niveaux ;
- classes ;
- sections ;
- années scolaires ;
- capacités ;
- responsables de classe ;
- enseignants affectés.

---

# Vérifier avant de créer une classe

Avant toute création, vérifier :

- que la classe n’existe pas déjà ;
- que son niveau existe ;
- que son nom suit la convention ;
- que l’année scolaire est correcte ;
- que la section est nécessaire ;
- que la direction a validé la structure.

## Exemple de cohérence

Éviter de mélanger :

```text
6e A
6ème A
Sixième A
6 A
```

Une convention unique facilite :

- la recherche ;
- les rapports ;
- les imports ;
- les affectations ;
- la communication.

---

# Modifier une classe

Une modification peut affecter plusieurs données.

Par exemple :

- changer le nom ;
- changer le niveau ;
- changer l’année scolaire ;
- supprimer une classe ;
- déplacer tous les élèves.

Ces actions doivent être réalisées avec prudence et parfois escaladées à RELAIS.

---

# Vérifier les listes de classe

Après les inscriptions, l’administrateur doit pouvoir :

- ouvrir la classe ;
- vérifier le nombre d’élèves ;
- identifier les élèves absents de la liste ;
- détecter les doublons ;
- vérifier les statuts ;
- comparer avec la liste officielle.

Les écarts doivent être expliqués avant le lancement.

---

# Exercice pratique 7

Le participant doit :

1. ouvrir une classe ;
2. comparer le nombre d’élèves avec une liste fournie ;
3. identifier un élève manquant ;
4. vérifier ses filtres ;
5. retrouver le dossier ;
6. expliquer la correction appropriée.

---

# 9. Gérer les utilisateurs

Selon le périmètre, un administrateur autorisé peut gérer :

- personnel administratif ;
- enseignants ;
- direction ;
- parents ;
- élèves ;
- comptes d’accès.

La gestion des utilisateurs est sensible, car elle détermine l’accès aux données.

---

# Avant de créer un utilisateur

Vérifier :

- l’identité ;
- l’adresse e-mail ou identifiant ;
- le rôle réel ;
- l’établissement ;
- la nécessité du compte ;
- l’autorisation ;
- l’absence de compte existant.

---

# Attribuer le bon rôle

Un rôle doit correspondre au travail réel.

Ne pas attribuer un rôle supérieur pour « faciliter » temporairement une action.

Une permission excessive peut exposer :

- les données des élèves ;
- les données financières ;
- les résultats scolaires ;
- les comptes ;
- les informations confidentielles.

---

# Principe du moindre privilège

L’utilisateur doit recevoir uniquement les permissions nécessaires à son travail.

Par exemple :

- un enseignant ne doit pas gérer tous les comptes ;
- un parent ne doit pas consulter les autres élèves ;
- un agent administratif ne doit pas accéder aux finances s’il n’en est pas responsable ;
- un utilisateur de test ne doit pas conserver un accès administrateur.

---

# Désactiver un compte

Lorsqu’une personne quitte l’établissement ou change de poste :

- vérifier la décision ;
- désactiver l’accès ;
- préserver l’historique ;
- réattribuer les responsabilités ;
- ne pas partager son ancien compte ;
- documenter la date.

Un compte inutilisé mais actif représente un risque.

---

# Réinitialiser un accès

L’administrateur doit suivre le processus prévu pour :

- vérifier l’identité ;
- réinitialiser le mot de passe ;
- transmettre l’accès de manière sécurisée ;
- demander le changement lors de la prochaine connexion ;
- ne jamais demander le mot de passe personnel de l’utilisateur.

---

# Exercice pratique 8

Scénario :

> Une nouvelle secrétaire rejoint l’établissement. Elle doit gérer les inscriptions, mais ne doit pas accéder aux informations financières.

Le participant doit :

1. vérifier si un compte existe ;
2. créer ou activer le compte ;
3. choisir le rôle approprié ;
4. vérifier ses permissions ;
5. tester un accès autorisé ;
6. tester qu’un accès financier lui est refusé.

---

# 10. Consulter les présences

Selon les permissions, l’administrateur peut :

- consulter les sessions ;
- rechercher une date ;
- rechercher une classe ;
- vérifier les absences ;
- consulter les retards ;
- identifier les sessions manquantes ;
- produire un rapport ;
- corriger certaines informations autorisées.

L’administrateur ne doit pas nécessairement saisir les présences à la place des enseignants.

---

# Vérifier une anomalie de présence

Avant de modifier :

- vérifier la classe ;
- vérifier la date ;
- identifier l’enseignant ;
- comprendre le statut ;
- vérifier la justification ;
- confirmer son autorisation.

Une absence ne doit pas être transformée en présence sans preuve ou validation.

---

# Sessions manquantes

Lorsqu’une classe ne possède pas de présence enregistrée :

- vérifier si la session était prévue ;
- contacter l’enseignant ;
- confirmer l’horaire ;
- distinguer un oubli d’un problème technique ;
- suivre la correction.

Le rôle de l’administrateur est de protéger la fiabilité du suivi, pas de masquer les oublis.

---

# 11. Consulter les données académiques

Selon le rôle, l’administrateur peut :

- consulter les évaluations ;
- vérifier que les classes sont configurées ;
- consulter les résultats ;
- produire un rapport ;
- vérifier les dossiers incomplets ;
- accompagner la génération des bulletins.

Il ne doit pas nécessairement :

- saisir les notes à la place des enseignants ;
- modifier une note sans procédure ;
- changer une règle de calcul ;
- publier des résultats non validés.

---

# Anomalies fréquentes

- élève absent d’une évaluation ;
- mauvaise classe ;
- matière incorrecte ;
- note manquante ;
- note impossible ;
- mauvaise échelle ;
- calcul inattendu ;
- bulletin incomplet.

L’administrateur doit distinguer :

- problème de configuration ;
- problème de donnée ;
- erreur d’utilisation ;
- défaut technique.

# 14. Vérifier son travail

Chaque opération importante doit se terminer par une vérification.

## Méthode RELAIS

```text
Identifier
    ↓
Effectuer l’action
    ↓
Enregistrer
    ↓
Rouvrir ou consulter
    ↓
Comparer avec la source
    ↓
Confirmer
```

---

# Questions de contrôle

Après une action, demander :

- Ai-je utilisé le bon dossier ?
- Suis-je dans la bonne année scolaire ?
- Ai-je sélectionné la bonne classe ?
- Les informations correspondent-elles au document ?
- L’enregistrement a-t-il réussi ?
- Le résultat apparaît-il au bon endroit ?
- Une autre personne doit-elle valider ?
- Ai-je modifié uniquement ce qui était nécessaire ?

---

# Contrôle à quatre points

Pour les actions importantes, vérifier :

1. **La personne**  
   Est-ce le bon élève, parent ou utilisateur ?

2. **La période**  
   Est-ce la bonne année, date ou période ?

3. **Le contexte**  
   Est-ce la bonne classe, matière ou catégorie ?

4. **Le résultat**  
   L’action apparaît-elle correctement après l’enregistrement ?

---

# 15. Corriger une erreur

Lorsqu’une erreur est découverte, l’administrateur doit éviter de paniquer ou de cacher le problème.

## Procédure

1. arrêter les nouvelles modifications liées ;
2. identifier l’information concernée ;
3. évaluer l’impact ;
4. conserver les preuves utiles ;
5. vérifier son niveau d’autorisation ;
6. corriger selon la procédure ;
7. demander une validation lorsque nécessaire ;
8. vérifier le résultat ;
9. documenter la correction ;
10. informer les personnes concernées.

---

# Ne pas multiplier les corrections

Lorsqu’un administrateur ne comprend pas l’origine d’une erreur, il ne doit pas essayer plusieurs changements au hasard.

Cela peut :

- aggraver le problème ;
- supprimer des informations ;
- rendre l’historique difficile à comprendre ;
- affecter d’autres modules.

Il faut demander de l’aide avec des informations précises.

---

# Exemple de signalement

## Signalement insuffisant

> La liste est fausse.

## Signalement exploitable

> Dans l’année scolaire 2026-2027, la classe 4e B affiche 39 élèves alors que la liste validée en contient 40. L’élève Awa Kaboré possède un dossier actif, mais son inscription apparaît dans la 4e A. Le problème a été observé le 28 juillet à 10 h 20 avec le compte administratif de Fatimata.

# 17. Savoir quand demander de l’aide

L’administrateur doit demander de l’aide lorsqu’il rencontre :

- un accès refusé inattendu ;
- un doublon difficile à confirmer ;
- une modification sensible ;
- un calcul incorrect ;
- un problème touchant plusieurs utilisateurs ;
- une exposition de données ;
- une opération financière erronée ;
- une action impossible à annuler ;
- une erreur technique persistante ;
- une décision qui dépasse son rôle.

Demander de l’aide n’est pas un échec.

C’est une mesure de protection.

# Critères de validation

Un administrateur peut être considéré comme opérationnel lorsqu’il :

- utilise son propre compte ;
- comprend les limites de son rôle ;
- recherche avant de créer ;
- identifie correctement les dossiers ;
- réalise les opérations essentielles ;
- vérifie chaque action importante ;
- évite les modifications non autorisées ;
- protège les informations ;
- sait corriger une erreur simple ;
- sait quand escalader ;
- peut travailler sans accompagnement permanent.

---

# Besoin de rattrapage

Un accompagnement complémentaire est nécessaire lorsque l’administrateur :

- crée fréquemment des doublons ;
- confond élève et inscription ;
- travaille dans la mauvaise année scolaire ;
- attribue des rôles excessifs ;
- ne vérifie pas ses opérations ;
- modifie des données sans source ;
- partage son compte ;
- ne sait pas identifier le bon dossier ;
- dépend constamment du formateur ;
- ne sait pas signaler un problème.

# Erreurs fréquentes

## Créer avant de rechercher

Cela produit des doublons.

---

## Confondre le dossier et l’inscription

Le dossier représente la personne.

L’inscription représente une année ou une période scolaire.

---

## Travailler dans la mauvaise année

L’action peut apparaître correcte, mais ne pas être visible dans la période active.

---

## Utiliser le compte d’un collègue

L’historique devient peu fiable et la sécurité diminue.

---

## Donner trop de permissions

Un rôle plus puissant n’est pas une solution à un problème de processus.

---

## Modifier sans source

Une demande orale ne suffit pas toujours pour changer une donnée sensible.

---

## Supprimer au lieu de désactiver

La suppression peut faire disparaître un historique nécessaire.

---

## Ne pas vérifier après l’enregistrement

Le message de confirmation ne garantit pas que les bonnes informations ont été choisies.

---

## Corriger au hasard

Plusieurs modifications non maîtrisées peuvent aggraver le problème.

---

## Faire le travail à la place des autres rôles

Un administrateur ne doit pas absorber toutes les responsabilités de l’établissement.

---

✔ Vérifier la source d’une modification.

✔ Distinguer l’élève de son inscription.

✔ Attribuer uniquement les permissions nécessaires.

✔ Vérifier après chaque opération importante.

✔ Documenter les corrections sensibles.

✔ Protéger les données.
