# KARMDA

# 07 - Formation continue et perfectionnement

## Méthode RELAIS

> Ce document présente la politique officielle de formation continue des utilisateurs de KARMDA.

**Objectif**

Maintenir et développer les compétences des utilisateurs tout au long de l'utilisation de KARMDA.

La formation n'est pas un événement.

C'est un processus continu.

---

# Pourquoi une formation continue ?

Les établissements évoluent.

Les équipes changent.

De nouveaux employés arrivent.

Le logiciel s'améliore.

Les procédures évoluent.

Une formation réalisée une seule fois ne suffit donc pas à garantir une utilisation durable de KARMDA.

---

# Notre philosophie

Chez RELAIS, la compétence n'est jamais considérée comme acquise définitivement.

Elle doit être entretenue.

Chaque nouvelle expérience permet de renforcer les bonnes pratiques et d'améliorer l'utilisation quotidienne du logiciel.

> **Une organisation qui apprend progresse plus vite qu'une organisation qui se contente de travailler.**

---

# Les objectifs

La formation continue permet de :

- maintenir les compétences ;
- accueillir les nouveaux utilisateurs ;
- présenter les nouvelles fonctionnalités ;
- corriger les mauvaises pratiques observées ;
- harmoniser les méthodes de travail ;
- améliorer la qualité des données.

---

# Quand organiser une formation ?

Une formation complémentaire peut être organisée :

- lors de l'arrivée d'un nouvel employé ;
- après une mise à jour importante de KARMDA ;
- lorsqu'une nouvelle fonctionnalité est déployée ;
- après un changement d'organisation ;
- lorsqu'un besoin particulier est identifié.

Une session de rappel peut également être organisée à intervalles réguliers.

---

# Les nouveaux utilisateurs

Chaque nouvel utilisateur doit suivre une formation adaptée à son rôle.

Même si ses collègues maîtrisent déjà KARMDA.

La transmission informelle entre collègues ne remplace pas une formation officielle.

---

# Les mises à jour du logiciel

Chaque évolution importante de KARMDA peut nécessiter :

- une présentation ;
- une démonstration ;
- une documentation mise à jour ;
- des exercices pratiques.

Les utilisateurs doivent comprendre non seulement **ce qui change**, mais aussi **pourquoi cela change**.

---

# Corriger les mauvaises habitudes

Avec le temps, certaines pratiques peuvent apparaître :

- oublis de vérification ;
- procédures incomplètes ;
- raccourcis inappropriés ;
- mauvaises interprétations.

La formation continue permet de corriger ces habitudes avant qu'elles ne deviennent des pratiques permanentes.

---

# Les utilisateurs référents

Chaque établissement est encouragé à identifier un ou plusieurs référents KARMDA.

Le référent :

- accompagne ses collègues ;
- répond aux questions courantes ;
- relaie les besoins auprès de RELAIS ;
- participe aux formations avancées.

Le référent ne remplace jamais le support RELAIS.

Il facilite son intervention.

---

# Les retours d'expérience

Chaque établissement accumule une expérience précieuse.

Les difficultés rencontrées.

Les solutions trouvées.

Les bonnes pratiques.

Ces retours permettent d'améliorer :

- les formations ;
- la documentation ;
- KARMDA lui-même.

Chaque retour est une opportunité de progresser.

---

# Le rôle de RELAIS

RELAIS accompagne les établissements en :

- mettant à jour la documentation ;
- proposant de nouvelles formations ;
- partageant les meilleures pratiques ;
- répondant aux questions ;
- accompagnant les évolutions du logiciel.

Notre objectif est de construire une relation durable avec chaque établissement.

---

# Les responsabilités de l'établissement

L'établissement s'engage à :

- intégrer les nouveaux utilisateurs ;
- participer aux formations nécessaires ;
- informer RELAIS des changements importants ;
- encourager les bonnes pratiques ;
- maintenir un haut niveau de qualité des données.

---

# Les responsabilités des utilisateurs

Chaque utilisateur est responsable de :

- maintenir ses compétences ;
- suivre les formations proposées ;
- signaler les difficultés rencontrées ;
- partager les bonnes pratiques ;
- respecter les procédures.

L'apprentissage est une responsabilité individuelle autant que collective.

---

# Les bénéfices

Une politique de formation continue permet :

- une meilleure qualité des données ;
- une meilleure autonomie des équipes ;
- moins d'erreurs ;
- une adoption durable du logiciel ;
- une meilleure satisfaction des utilisateurs.

---

# Exercices de perfectionnement

Exemples :

- reprendre un scénario complet d'inscription ;
- créer une nouvelle évaluation ;
- résoudre une anomalie ;
- analyser un tableau de bord ;
- préparer un bulletin.

Ces exercices entretiennent les automatismes.

---

# Checklist annuelle

- [ ] Former les nouveaux utilisateurs.
- [ ] Organiser une session de rappel.
- [ ] Présenter les nouveautés de KARMDA.
- [ ] Identifier les difficultés récurrentes.
- [ ] Mettre à jour les supports internes.
- [ ] Désigner ou confirmer les référents.
- [ ] Recueillir les retours des utilisateurs.

---

# Erreurs fréquentes

- considérer la formation comme terminée après le déploiement ;
- négliger les nouveaux utilisateurs ;
- conserver des procédures obsolètes ;
- ignorer les nouvelles fonctionnalités ;
- ne pas partager les bonnes pratiques.

---

# Bonnes pratiques RELAIS

✔ Former régulièrement les équipes.

✔ Valoriser les utilisateurs référents.

✔ Partager les retours d'expérience.

✔ Mettre à jour les connaissances.

✔ Corriger rapidement les mauvaises habitudes.

✔ Favoriser une culture d'apprentissage.

✔ Faire évoluer les pratiques avec le logiciel.

---

# Ce qu'il faut retenir

- La formation est un processus continu.
- Les compétences doivent être entretenues.
- Chaque évolution du logiciel est une occasion d'apprendre.
- Les référents jouent un rôle essentiel dans l'adoption.
- Les retours des utilisateurs contribuent à améliorer KARMDA.

---

## Phrase à retenir

> **« La meilleure version de KARMDA n'est pas seulement celle que nous développons. C'est celle que les utilisateurs apprennent à maîtriser ensemble. »**

# Cas pratique

Un établissement utilise KARMDA depuis deux ans. Plusieurs nouveaux enseignants ont été recrutés, certaines fonctionnalités ont évolué et des erreurs récurrentes apparaissent dans la gestion des présences et des évaluations. Les utilisateurs expérimentés pensent qu'une nouvelle formation serait une perte de temps.

Expliquez :

- pourquoi une session de formation continue serait pertinente ;
- quelles catégories d'utilisateurs devraient y participer ;
- quels sujets devraient être abordés en priorité ;
- comment les référents pourraient contribuer à son succès ;
- en quoi cette démarche améliorerait durablement la qualité des données et l'utilisation de KARMDA.

---

# Conclusion

La réussite d'un projet numérique ne dépend pas uniquement de la qualité du logiciel.

Elle dépend de la capacité des femmes et des hommes qui l'utilisent à progresser avec lui.

Chez RELAIS, nous considérons qu'un déploiement est réussi lorsque les équipes deviennent autonomes.

Nous considérons qu'il est durable lorsqu'elles continuent à apprendre.

**Vous avancez quand on prend le relais.**
