# KARMDA

# 03 - Formation des enseignants

## Formation RELAIS

> Ce document présente le programme officiel de formation des enseignants utilisant KARMDA.

**Objectif**

À la fin de cette formation, chaque enseignant sera capable d'utiliser KARMDA pour gérer ses classes, enregistrer les présences, créer des évaluations, saisir les notes, consulter les résultats et produire des données académiques fiables.

**Durée indicative**

2 à 3 heures

**Public concerné**

- Enseignants
- Professeurs principaux
- Responsables pédagogiques
- Enseignants remplaçants
- Coordinateurs pédagogiques

**Prérequis**

- Compte KARMDA actif
- Affectation à au moins une classe
- Participation à l'onboarding
- Connaissance des règles académiques de l'établissement

---

# Pourquoi cette formation est importante

Chaque jour, les enseignants produisent les informations académiques les plus importantes de l'établissement.

Par exemple :

- les présences ;
- les absences ;
- les retards ;
- les évaluations ;
- les notes ;
- les appréciations ;
- les observations.

Ces informations servent ensuite à :

- calculer les moyennes ;
- produire les bulletins ;
- suivre les performances ;
- informer les parents ;
- prendre des décisions pédagogiques ;
- constituer l'historique scolaire des élèves.

Une erreur d'un seul enseignant peut donc affecter :

- un élève ;
- une classe ;
- un bulletin ;
- un conseil de classe ;
- un classement ;
- un rapport officiel.

---

# Notre philosophie

L'enseignant n'utilise pas KARMDA pour remplir un logiciel.

Il utilise KARMDA pour produire une histoire académique fiable.

Chaque présence enregistrée.

Chaque absence.

Chaque note.

Chaque appréciation.

Devient une partie permanente du parcours scolaire de l'élève.

> **Une note n'est pas une opinion. C'est une information académique.**

---

# Les responsabilités de l'enseignant

L'enseignant est responsable de :

- ses classes ;
- ses séances de présence ;
- ses évaluations ;
- ses notes ;
- ses appréciations ;
- la qualité des données qu'il produit.

Il n'est généralement pas responsable :

- des comptes utilisateurs ;
- des paramètres ;
- des inscriptions ;
- des structures académiques ;
- des permissions ;
- des informations financières.

Chaque rôle possède ses responsabilités.

---

# Résultats attendus

À la fin de cette formation, l'enseignant doit pouvoir :

- se connecter ;
- retrouver ses classes ;
- enregistrer une présence ;
- corriger une présence autorisée ;
- créer une évaluation ;
- saisir les notes ;
- vérifier les résultats ;
- consulter les moyennes ;
- identifier une anomalie ;
- demander de l'aide correctement.

# 1. Se connecter

Chaque enseignant utilise son propre compte.

Jamais celui :

- d'un collègue ;
- d'un directeur ;
- d'un administrateur.

Pourquoi ?

Parce que KARMDA doit toujours savoir :

- qui a enregistré une présence ;
- qui a saisi une note ;
- qui a modifié une évaluation.

Le partage de compte supprime cette traçabilité.

---

# 2. Comprendre son espace

L'enseignant ne voit que les informations correspondant à son rôle.

Il retrouve généralement :

- ses classes ;
- ses matières ;
- ses évaluations ;
- ses présences ;
- ses résultats.

Il ne doit pas s'inquiéter de ne pas voir :

- toutes les classes ;
- tous les élèves ;
- tous les enseignants.

C'est normal.

Les permissions protègent les données.

---

# 3. Retrouver ses classes

Avant chaque cours :

- vérifier la classe ;
- vérifier la matière ;
- vérifier la date ;
- vérifier l'année scolaire.

Une erreur de classe peut produire des présences ou des notes dans le mauvais groupe.

---

# Exercice

Retrouver :

- votre classe de 6e A ;
- votre classe de 5e B ;
- votre emploi du temps du jour.

---

# 4. Enregistrer les présences

La présence doit être enregistrée aussi proche que possible du début du cours.

Pourquoi ?

Parce que la mémoire n'est pas fiable.

Attendre plusieurs heures augmente le risque d'erreur.

---

## Avant de commencer

Vérifier :

- la classe ;
- la matière ;
- la date ;
- l'heure.

---

## Pour chaque élève

Choisir le statut approprié :

- Présent
- Absent
- Retard
- Absence justifiée
- Excusé
  (selon les règles de l'établissement)

---

## Après l'enregistrement

Toujours vérifier :

- le nombre d'élèves ;
- les absents ;
- les retards ;
- le message de confirmation.

> **Une présence n'est pas terminée lorsqu'elle est enregistrée. Elle est terminée lorsqu'elle est vérifiée.**

---

# Erreurs fréquentes

- mauvaise date ;
- mauvaise classe ;
- oubli d'enregistrer ;
- oubli de vérifier ;
- enregistrer le lendemain de mémoire.

---

# 5. Corriger une présence

Toutes les corrections ne sont pas autorisées.

Avant toute modification :

- vérifier la raison ;
- vérifier l'autorisation ;
- comprendre l'impact.

Une présence ne doit jamais être modifiée simplement parce qu'un élève l'affirme.

---

# 6. Créer une évaluation

Une évaluation représente un travail pédagogique précis.

Elle doit comporter :

- un titre clair ;
- une date ;
- une matière ;
- une classe ;
- un barème ;
- une période.

Exemples :

- Devoir 1
- Contrôle Chapitre 3
- Composition du premier trimestre

Éviter :

```
Test
Contrôle
Note
```

sans précision.

---

# Vérifier avant publication

Avant d'enregistrer :

- la classe ;
- la matière ;
- la date ;
- le barème ;
- les élèves concernés.

---

# 7. Saisir les notes

Avant toute saisie :

Les copies doivent être corrigées.

KARMDA n'est pas un brouillon.

C'est le registre officiel.

---

## Pendant la saisie

Vérifier :

- le bon élève ;
- la bonne évaluation ;
- le bon barème.

---

## Après la saisie

Comparer :

- la copie ;
- la note saisie ;
- le total des notes.

---

> **Une note saisie sans vérification est une erreur en attente d'être découverte.**

---

# Les notes ne se devinent pas

Ne jamais :

- arrondir "par habitude" ;
- augmenter une note "par sympathie" ;
- diminuer une note "pour punir" ;
- estimer une copie non corrigée.

Une note doit toujours être justifiable.

---

# Modifier une note

Avant modification :

Se demander :

Pourquoi ?

La réponse doit être objective.

Exemples :

- erreur de saisie ;
- erreur de calcul ;
- copie recorrigée ;
- procédure officielle.

Jamais :

- parce qu'un parent insiste ;
- parce qu'un élève insiste.

---

# 8. Consulter les résultats

Les résultats permettent :

- suivre une classe ;
- identifier les difficultés ;
- préparer un conseil de classe.

Ils ne doivent jamais être utilisés pour humilier un élève publiquement.

---

# 9. Comprendre les moyennes

Une moyenne est le résultat :

des évaluations,

pas de l'impression générale du professeur.

Lorsque la moyenne semble étrange :

vérifier :

- les notes ;
- les coefficients ;
- les absences ;
- les évaluations manquantes.

Avant de conclure à une erreur du système.

---

# 10. Préparer les bulletins

Avant publication :

Vérifier :

- toutes les notes ;
- les appréciations ;
- les absences ;
- les évaluations.

Le bulletin représente un document officiel.

---

# Les appréciations

Une appréciation doit être :

- professionnelle ;
- factuelle ;
- utile ;
- respectueuse.

Éviter :

```
Élève paresseux.

N'écoute jamais.

Catastrophique.
```

Préférer :

```
Participation irrégulière.

Des efforts plus constants permettraient une progression.

Bonne implication.
```

Le bulletin doit aider l'élève à progresser.

---

# 11. Confidentialité

Les enseignants ont accès à certaines données.

Ils ne doivent pas :

- montrer les notes d'une classe à une autre ;
- partager des captures d'écran ;
- utiliser le compte d'un collègue ;
- publier des bulletins.

Les informations scolaires appartiennent à l'établissement.

---

# 12. Vérifier son travail

Avant de quitter KARMDA :

Toujours vérifier :

- les présences ;
- les notes ;
- les appréciations.

La vérification fait partie du travail.

---

# Méthode RELAIS

```text
Préparer
    ↓
Enregistrer
    ↓
Vérifier
    ↓
Corriger si nécessaire
    ↓
Confirmer
```

- [ ] Enregistrer les présences.

- [ ] Corriger les copies.

- [ ] Vérifier les copies.

- [ ] Vérifier les notes.

- [ ] Contrôler les opérations du jour.

# Erreurs fréquentes

- enregistrer dans la mauvaise classe ;
- oublier une présence ;
- créer deux évaluations identiques ;
- saisir une note dans la mauvaise évaluation ;
- modifier une note sans justification ;
- oublier de vérifier ;
- publier un bulletin incomplet.

---

# Bonnes pratiques RELAIS

✔ Utiliser uniquement son compte.

✔ Enregistrer les présences immédiatement.

✔ Vérifier chaque évaluation.

✔ Corriger les copies avant la saisie.

✔ Contrôler les notes après l'enregistrement.

✔ Respecter la confidentialité.

✔ Demander de l'aide lorsqu'une anomalie apparaît.

✔ Produire des données fiables.

---

# Ce qu'il faut retenir

- Les présences doivent être enregistrées rapidement.
- Les évaluations doivent être préparées soigneusement.
- Les notes doivent être justifiées.
- Les bulletins sont des documents officiels.
- Les données produites aujourd'hui resteront dans l'histoire scolaire de l'élève.
- Vérifier est une étape obligatoire.

---

## Phrase à retenir

> **« Les élèves oublient parfois un cours. L'établissement ne doit jamais oublier son histoire académique. Chaque présence et chaque note enregistrées aujourd'hui deviennent une partie durable de cette histoire. »**

5. Pourquoi vérifier après chaque saisie ?
6. Quel est le rôle des appréciations ?
7. Pourquoi protéger les données scolaires ?

## Question de réflexion

Un parent vous appelle le soir même.

Son enfant a obtenu **9/20**.

Il affirme que son fils mérite **12/20**, car il est habituellement un bon élève. Il vous demande de modifier directement la note dans KARMDA pour éviter qu'il échoue au trimestre.

Expliquez :

- pourquoi vous ne pouvez pas modifier immédiatement la note ;
- quelles vérifications doivent être réalisées ;
- quelles procédures de l'établissement doivent être respectées ;
- dans quels cas une modification serait légitime ;
- pourquoi la confiance dans un système académique dépend de l'intégrité des données qu'il contient.

## Rappel:
