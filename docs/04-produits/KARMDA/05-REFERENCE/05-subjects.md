# KARMDA

# 05 - Matières

## Référence fonctionnelle

> Le module Matières permet de gérer le catalogue des matières enseignées dans l'établissement.

---

# Objectif

Le module Matières centralise les disciplines enseignées au sein de l'établissement.

Chaque matière constitue une référence utilisée dans plusieurs modules de KARMDA, notamment les classes, les évaluations, les notes et les bulletins.

---

# Utilisateurs concernés

- Administrateur
- Personnel administratif

---

# Permissions requises

Selon son rôle, un utilisateur peut :

- consulter les matières ;
- créer une matière ;
- modifier une matière ;
- supprimer une matière (selon les règles métier).

---

# Vue d'ensemble

Le module est composé de deux espaces.

## Création d'une matière

Chaque matière est créée à partir de :

- son nom ;
- son code.

Exemples :

| Nom                               | Code |
| --------------------------------- | ---- |
| Mathématiques                     | MATH |
| Français                          | FR   |
| Sciences de la Vie et de la Terre | SVT  |

---

## Catalogue des matières

Le registre présente toutes les matières disponibles dans l'établissement.

Pour chaque matière sont affichés :

- le nom ;
- le code.

Chaque ligne permet de modifier ou supprimer la matière.

---

# Fonctionnement

## Créer une matière

Créer une nouvelle matière en renseignant :

- son nom ;
- son code.

---

## Modifier une matière

Le nom ou le code peuvent être mis à jour selon les besoins de l'établissement.

---

## Supprimer une matière

Une matière peut être supprimée lorsqu'elle n'est plus utilisée et que cette suppression respecte les règles métier.

---

# Règles métier

## Une matière est créée une seule fois

Une matière est enregistrée une seule fois dans le catalogue.

Elle peut ensuite être associée à plusieurs classes.

---

## Le code identifie la matière

Le code facilite l'identification rapide de la matière dans l'ensemble du système.

Il est recommandé qu'il soit unique au sein de l'établissement.

---

## Une matière peut être utilisée par plusieurs classes

La même matière peut être enseignée dans plusieurs classes.

Il n'est donc pas nécessaire de créer plusieurs fiches identiques.

---

## Les matières sont réutilisées

Une matière créée dans ce module peut être utilisée dans :

- les classes ;
- les évaluations ;
- les notes ;
- les bulletins ;
- les rapports académiques.

---

# Cas d'utilisation

### Début d'année scolaire

Créer le catalogue des matières avant de les affecter aux classes.

---

### Nouvelle discipline

Ajouter une nouvelle matière lorsqu'elle est introduite dans le programme pédagogique.

---

# Erreurs fréquentes

- créer plusieurs fois la même matière ;
- utiliser des codes incohérents ;
- supprimer une matière encore utilisée dans une classe.

---

# Questions fréquentes

## Une matière peut-elle être utilisée dans plusieurs classes ?

Oui.

Une seule fiche suffit.

---

## Peut-on modifier le nom d'une matière ?

Oui, selon les permissions accordées.

---

## Pourquoi utiliser un code ?

Le code facilite l'identification et l'intégration de la matière dans les autres modules.

---

# Bonnes pratiques RELAIS

✔ Définir un code court et explicite.

✔ Éviter les doublons.

✔ Créer le catalogue avant la configuration des classes.

✔ Utiliser une nomenclature cohérente dans tout l'établissement.

✔ Vérifier les dépendances avant de supprimer une matière.

---

# Modules associés

- Classes
- Notes
- Carnet de notes
- Bulletins
- Emploi du temps

---

# Ce qu'il faut retenir

- Une matière est créée une seule fois.
- Le catalogue est partagé par tout l'établissement.
- Les matières alimentent plusieurs modules de KARMDA.
- Une bonne organisation du catalogue simplifie toute la gestion académique.

---

## Phrase à retenir

> **« Une matière n'appartient pas à une classe. Elle appartient au programme pédagogique de l'établissement. »**
