# KARMDA

# 05 - Référence fonctionnelle

## Documentation officielle RELAIS

> Cette section constitue la documentation de référence officielle de KARMDA. Elle décrit le fonctionnement des différents modules du logiciel ainsi que les règles métier qui les gouvernent.

---

# Objectif

Cette documentation permet de comprendre en détail le fonctionnement de KARMDA.

Elle s'adresse aux utilisateurs qui souhaitent approfondir un module particulier après leur formation.

Contrairement aux guides de formation, cette documentation ne suit pas un parcours pédagogique.

Chaque document peut être consulté indépendamment selon le besoin.

---

# À qui s'adresse cette documentation ?

Cette documentation est destinée à :

- Administrateurs
- Chefs d'établissement
- Enseignants
- Personnel administratif
- Support RELAIS
- Équipe de déploiement
- Développeurs
- Auditeurs fonctionnels

Chaque lecteur peut consulter uniquement les modules correspondant à ses responsabilités.

---

# Philosophie de la documentation

Les guides de formation expliquent **comment utiliser** KARMDA.

La documentation de référence explique **comment fonctionne** KARMDA.

Chaque module répond à quatre questions essentielles :

1. Pourquoi ce module existe-t-il ?
2. Que permet-il de faire ?
3. Quelles règles métier applique-t-il ?
4. Avec quels autres modules interagit-il ?

Notre objectif n'est pas seulement de décrire des écrans.

Nous expliquons également les principes fonctionnels qui permettent à l'établissement de comprendre les décisions prises par le logiciel.

---

# Structure des documents

Afin de garantir une documentation cohérente, tous les modules suivent la même organisation.

Chaque document comprend généralement les sections suivantes :

- Objectif du module
- Utilisateurs concernés
- Permissions requises
- Vue d'ensemble
- Concepts principaux
- Règles métier
- Fonctionnement
- Champs importants
- États possibles
- Exemples
- Erreurs fréquentes
- Questions fréquentes
- Modules associés

Cette structure facilite la recherche d'information et assure une expérience homogène dans toute la documentation.

---

# Ce que cette documentation n'est pas

Cette documentation n'est pas :

- un guide d'installation ;
- un manuel de formation ;
- une documentation technique du code source ;
- une procédure de déploiement.

Ces sujets sont traités dans les sections correspondantes de la documentation RELAIS.

---

# Les règles métier

KARMDA applique un ensemble de règles métier destinées à garantir :

- la cohérence des données ;
- la sécurité des informations ;
- la traçabilité des opérations ;
- l'intégrité des dossiers scolaires.

Certaines opérations peuvent donc être limitées selon :

- le rôle de l'utilisateur ;
- l'année scolaire ;
- le statut d'un élève ;
- les paramètres de l'établissement.

Ces restrictions sont volontaires et contribuent à la fiabilité du système.

---

# Les permissions

Toutes les fonctionnalités de KARMDA sont protégées par un système de permissions.

Selon son rôle, un utilisateur peut :

- consulter ;
- créer ;
- modifier ;
- supprimer ;
- approuver certaines opérations.

Les permissions garantissent que chaque utilisateur n'accède qu'aux informations nécessaires à son travail.

---

# Les conventions utilisées

Afin de rendre cette documentation claire et cohérente, plusieurs conventions sont utilisées.

### Informations importantes

Les informations ayant un impact sur les données ou le fonctionnement du système sont clairement mises en évidence.

### Exemples

Les exemples illustrent des situations réelles rencontrées dans les établissements.

Ils servent à faciliter la compréhension des concepts.

### Bonnes pratiques

Les bonnes pratiques présentent les recommandations officielles de RELAIS.

Elles permettent d'utiliser KARMDA de manière efficace et sécurisée.

### Règles métier

Les règles métier expliquent pourquoi certaines actions sont autorisées ou interdites dans le logiciel.

Comprendre ces règles est essentiel pour bien utiliser KARMDA.

---

# Mise à jour de la documentation

KARMDA évolue régulièrement.

Cette documentation est donc mise à jour afin de refléter les évolutions du logiciel.

Chaque nouvelle version peut introduire :

- de nouvelles fonctionnalités ;
- des améliorations ;
- des changements de procédures ;
- de nouvelles règles métier.

Les établissements sont encouragés à consulter régulièrement la documentation.

# Les principes RELAIS

Toute la documentation de référence repose sur cinq principes.

## Comprendre avant d'utiliser

Un utilisateur qui comprend les règles métier travaille plus efficacement.

---

## Expliquer les décisions du logiciel

Chaque comportement important de KARMDA doit pouvoir être compris.

Le logiciel ne doit jamais sembler arbitraire.

---

## Garantir la cohérence

Tous les modules utilisent les mêmes principes de fonctionnement afin d'assurer une expérience homogène.

---

## Favoriser l'autonomie

La documentation doit permettre aux utilisateurs de répondre eux-mêmes à la majorité de leurs questions.

---

## Améliorer en continu

La documentation évolue avec KARMDA.

Les retours des établissements permettent d'enrichir continuellement ces guides.

---

# Ce qu'il faut retenir

- Cette documentation constitue la référence officielle de KARMDA.
- Chaque module peut être consulté indépendamment.
- Les règles métier expliquent le fonctionnement du logiciel.
- Les permissions protègent les données.
- Les bonnes pratiques permettent une utilisation fiable et durable.
- La documentation évolue avec le produit.

---

## Phrase à retenir

> **« Un bon logiciel permet d'accomplir une tâche. Une bonne documentation permet de comprendre pourquoi cette tâche fonctionne ainsi. »**

# Conclusion

La documentation de référence accompagne les utilisateurs tout au long de la vie de KARMDA.

Elle constitue la mémoire fonctionnelle du logiciel.

Chez RELAIS, nous considérons qu'une fonctionnalité n'est réellement terminée que lorsqu'elle peut être comprise, utilisée et expliquée.

**Vous avancez quand on prend le relais.**
