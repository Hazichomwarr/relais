# KARMDA

# 01 - Tableau de bord

## Référence fonctionnelle

> Le tableau de bord constitue le point d'entrée principal de KARMDA. Il présente une vue synthétique des informations les plus importantes selon le rôle de l'utilisateur.

---

# Objectif

Le tableau de bord permet à chaque utilisateur d'obtenir, dès sa connexion, une vision rapide de la situation actuelle de son établissement ou de ses activités.

Son rôle est d'aider l'utilisateur à identifier :

- ce qui mérite son attention ;
- les actions à réaliser ;
- les indicateurs importants ;
- les événements récents.

Le tableau de bord ne remplace jamais les modules spécialisés.

Il en présente une synthèse.

---

# Utilisateurs concernés

Le tableau de bord est disponible pour tous les utilisateurs autorisés.

Son contenu varie selon le rôle :

- Super administrateur
- Administrateur
- Personnel administratif
- Enseignant
- Parent
- Élève

Chaque utilisateur voit uniquement les informations correspondant à ses permissions.

---

# Permissions requises

L'accès au tableau de bord nécessite :

- une authentification valide ;
- un compte actif ;
- un établissement sélectionné lorsque cela est applicable.

Les indicateurs affichés respectent toujours les permissions de l'utilisateur.

---

# Vue d'ensemble

Le tableau de bord est organisé en plusieurs zones.

## Informations générales

En haut de la page sont affichés :

- le nom de l'établissement ;
- le titre du tableau de bord ;
- la date du jour.

Ces informations permettent de situer immédiatement le contexte de travail.

---

## Cartes de synthèse (KPIs)

Les premières cartes présentent les principaux indicateurs de l'établissement.

Exemples :

- Élèves actifs
- Classes actives
- Enseignants actifs

Chaque carte affiche :

- la valeur actuelle ;
- une courte description.

Ces indicateurs donnent une vision rapide de la structure de l'établissement.

---

## Widgets opérationnels

Le tableau de bord présente ensuite plusieurs widgets permettant de suivre les activités quotidiennes.

Selon les modules activés, on peut notamment retrouver :

### Présences du jour

Ce widget affiche :

- le nombre d'élèves présents ;
- les absents ;
- les retards ;
- les élèves excusés ;
- le pourcentage de progression de la saisie.

Il permet également d'accéder rapidement à la saisie des présences.

---

### Cours du jour

Ce widget présente :

- les cours prévus ;
- les classes concernées ;
- les actions rapides associées.

Lorsqu'aucun cours n'est prévu, le widget l'indique clairement.

---

### Paiements reçus aujourd'hui

Ce widget affiche :

- le montant total encaissé ;
- le nombre de paiements enregistrés.

Il permet un accès direct au module Finance.

---

### Évaluations à venir

Ce widget informe les utilisateurs des évaluations programmées dans les prochains jours.

Il facilite la préparation pédagogique.

---

### Activité récente

Cette section présente les dernières opérations réalisées dans l'établissement.

Par exemple :

- présence enregistrée ;
- paiement effectué ;
- élève ajouté.

Chaque activité indique :

- la nature de l'action ;
- la date ;
- l'heure.

Cette vue améliore la traçabilité des opérations.

---

# Actions rapides

Plusieurs widgets proposent des raccourcis permettant d'accéder directement aux modules concernés.

Exemples :

- Saisir les présences
- Voir les classes
- Voir les paiements
- Voir les évaluations

Ces raccourcis permettent de gagner du temps.

Ils ne remplacent pas les fonctionnalités complètes des modules.

---

# Actualisation des données

Les informations affichées reflètent l'état actuel des données enregistrées dans KARMDA.

Les indicateurs évoluent automatiquement au fur et à mesure des opérations réalisées par les utilisateurs.

Certaines informations peuvent nécessiter un rechargement de la page selon la configuration technique de l'établissement.

---

# Règles métier

Le tableau de bord respecte plusieurs principes fondamentaux.

## Les données sont en lecture seule

Les indicateurs ne permettent pas de modifier directement les données.

Les modifications sont réalisées dans les modules concernés.

---

## Les widgets sont des synthèses

Chaque widget présente uniquement les informations essentielles.

Pour consulter le détail complet, l'utilisateur doit accéder au module correspondant.

---

## Les informations respectent les permissions

Deux utilisateurs ayant des rôles différents peuvent consulter des tableaux de bord différents.

Aucune information non autorisée n'est affichée.

---

## Les indicateurs dépendent des données disponibles

Lorsqu'aucune donnée n'existe, le tableau de bord affiche un état vide plutôt qu'une information erronée.

---

# Cas d'utilisation

## Début de journée

Le directeur consulte :

- les présences ;
- les cours programmés ;
- les paiements du jour.

Il identifie immédiatement les priorités.

---

## Suivi administratif

Le personnel administratif vérifie :

- les nouvelles inscriptions ;
- les paiements ;
- les activités récentes.

---

## Contrôle quotidien

Avant de quitter l'établissement, le responsable consulte :

- les présences enregistrées ;
- les opérations récentes ;
- les éventuelles anomalies.

---

# Erreurs fréquentes

- considérer le tableau de bord comme un rapport complet ;
- oublier d'ouvrir les modules pour consulter le détail ;
- interpréter un indicateur sans tenir compte de son contexte ;
- penser que tous les utilisateurs voient le même tableau de bord.

---

# Questions fréquentes

## Pourquoi certaines cartes affichent-elles zéro ?

Parce qu'aucune donnée n'a encore été enregistrée ou que l'utilisateur ne possède pas les permissions nécessaires.

---

## Pourquoi un autre utilisateur voit-il un tableau de bord différent ?

Le contenu dépend du rôle et des permissions.

---

## Les indicateurs sont-ils modifiables ?

Non.

Ils reflètent les données enregistrées dans les modules.

---

# Bonnes pratiques RELAIS

✔ Commencer chaque journée par le tableau de bord.

✔ Utiliser les widgets pour identifier les priorités.

✔ Accéder aux modules pour effectuer les opérations.

✔ Vérifier régulièrement les activités récentes.

✔ Ne jamais prendre une décision importante sur la seule base d'un indicateur sans consulter les détails.

- Le tableau de bord facilite la prise de décision quotidienne.
