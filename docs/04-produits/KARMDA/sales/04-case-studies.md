# Pourquoi ce document est important

Les établissements scolaires prennent rarement une décision uniquement sur la base d'une liste de fonctionnalités.

Ils veulent savoir comment KARMDA peut améliorer leur quotidien.

Une étude de cas permet de raconter une situation réelle, un problème rencontré et la manière dont KARMDA a apporté une solution.

Les histoires sont plus faciles à retenir que les arguments techniques.

---

# Comment utiliser une étude de cas

Une bonne étude de cas suit toujours la même structure.

```
Situation

↓

Problème

↓

Solution

↓

Résultat
```

N'ajoutez pas d'éléments inutiles.

Restez simple.

---

# Cas n°1

## Préparation des bulletins

### Situation

Un établissement accueille environ 450 élèves.

Chaque fin de trimestre, les enseignants transmettent leurs notes à l'administration.

### Problème

Les moyennes sont calculées manuellement.

La préparation des bulletins prend plusieurs jours.

Des erreurs de calcul apparaissent régulièrement.

### Solution

Les enseignants saisissent directement leurs notes dans KARMDA.

Les calculs sont automatisés selon les règles définies par l'établissement.

### Résultat

- moins d'erreurs ;
- préparation plus rapide ;
- bulletins disponibles plus tôt ;
- administration soulagée.

---

# Cas n°2

## Gestion des absences

### Situation

Les présences sont enregistrées sur papier.

### Problème

L'administration découvre parfois une absence plusieurs heures après le début des cours.

Les parents sont difficilement informés.

### Solution

Les enseignants enregistrent les présences directement dans KARMDA.

Les informations deviennent immédiatement disponibles.

### Résultat

- meilleure réactivité ;
- suivi simplifié ;
- communication améliorée avec les familles.

---

# Cas n°3

## Communication avec les parents

### Situation

Les annonces sont affichées sur un tableau ou envoyées individuellement.

### Problème

Certaines familles ne reçoivent jamais les informations.

### Solution

Les communications sont centralisées dans KARMDA.

Les parents disposent d'un point d'accès unique.

### Résultat

- informations mieux diffusées ;
- moins d'oublis ;
- relation école-famille renforcée.

---

# Cas n°4

## Vision de la direction

### Situation

La direction souhaite connaître rapidement les principaux indicateurs de l'établissement.

### Problème

Les informations sont réparties dans plusieurs registres et fichiers.

Chaque rapport demande plusieurs heures de travail.

### Solution

KARMDA centralise les données et propose des tableaux de bord.

### Résultat

- décisions plus rapides ;
- meilleure visibilité ;
- gain de temps pour la direction.

---

# Cas n°5

## Nouvelle année scolaire

### Situation

L'établissement prépare une nouvelle rentrée.

### Problème

Les classes, les enseignants et les inscriptions doivent être recréés manuellement.

### Solution

KARMDA accompagne la préparation de la nouvelle année scolaire grâce à une organisation centralisée.

### Résultat

- préparation plus rapide ;
- moins d'erreurs ;
- rentrée plus sereine.

---

# Comment raconter une étude de cas

Ne récitez jamais le texte.

Racontez l'histoire.

Par exemple :

> Une école que nous accompagnons rencontrait exactement cette difficulté. Les bulletins demandaient plusieurs jours de préparation et les erreurs étaient fréquentes. Après la mise en place de KARMDA, les enseignants saisissaient directement leurs notes et les calculs étaient automatisés. L'administration gagnait un temps considérable.

Une histoire inspire davantage confiance qu'une liste de fonctionnalités.

---

# Les erreurs à éviter

Ne jamais :

- exagérer les résultats ;
- inventer des chiffres ;
- citer un établissement sans son autorisation ;
- promettre des résultats garantis ;
- comparer négativement un concurrent.

Restez toujours honnête.

---

# Notre philosophie

Chaque établissement est unique.

Les études de cas servent à illustrer des situations comparables.

Elles ne constituent jamais une promesse de résultat identique.

Notre rôle est d'accompagner chaque école selon son contexte.

---

# Ce qu'il faut retenir

À l'issue de ce chapitre, retenez les idées suivantes :

- Une étude de cas raconte une histoire.
- Les histoires sont plus convaincantes que les fonctionnalités.
- Une étude de cas suit une structure simple : Situation → Problème → Solution → Résultat.
- Les exemples doivent rester réalistes et honnêtes.
- Les études de cas servent à aider le client à se projeter.
